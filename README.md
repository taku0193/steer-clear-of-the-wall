# steer-clear-of-the-wall

カメラの前で体を動かし、画面奥から迫る壁の安全領域にアバターを合わせて避ける
ブラウザゲームです。カメラ映像をゲーム画面の主表示にはせず、MediaPipeで検出した
姿勢をCanvas上のアバターとして描画します。カメラなしで動作を確認できるモック姿勢
モードも用意しています。

## 動作環境

- Ubuntuサーバーなど、Node.jsを実行できる環境
- Node.js 22.12以降を推奨
- npm
- カメラとWebAssemblyに対応したモダンブラウザ
- MediaPipeのWASMとモデルを取得できるインターネット接続

## セットアップ

リポジトリのルートで依存関係をインストールします。`sudo`は使用しません。

```bash
npm install
```

## 確認コマンド

```bash
# TypeScriptの型チェック
npm run typecheck

# プロダクションビルド
npm run build

# プロダクションサーバー
npm run start

# ゲームロジックのユニットテスト
npm test
```

ユニットテストは、当たり判定、スコア計算、壁パターン、ゲーム状態の初期化、
ゲームループの純粋ロジックを対象にします。カメラ、MediaPipeの実推論、Canvas、
ブラウザ操作は実機で確認します。

## 開発サーバーの起動

```bash
npm run dev
```

標準ではNext.jsがポート`3000`で待ち受けます。表示されたURLをブラウザで開いてください。

## SSH接続先で開発する場合

ブラウザのカメラAPIは安全なコンテキストでのみ利用できます。次のいずれかが必要です。

- ブラウザから`http://localhost`としてアクセスする
- HTTPSで配信する

SSH先サーバーのIPアドレスを通常のHTTPで直接開くと、カメラを利用できない場合が
あります。ローカルPCのターミナルからSSHポートフォワーディングを作成してください。

```bash
ssh -L 3000:127.0.0.1:3000 <ユーザー名>@<SSHサーバー>
```

SSH先で開発サーバーを起動します。

```bash
cd <リポジトリのパス>
npm run dev
```

ローカルPCのブラウザで次を開きます。

```text
http://localhost:3000
```

ローカルのポート`3000`が使用中の場合は、例えば
`-L 3001:127.0.0.1:3000`として、`http://localhost:3001`を開きます。

## カメラ利用条件

- `localhost`またはHTTPSで開く必要があります。
- ブラウザのカメラ許可ダイアログで使用を許可する必要があります。
- 音声は取得しません。
- MediaPipeのWASMと姿勢検出モデルを外部配信元から読み込むため、ブラウザから
  `cdn.jsdelivr.net`と`storage.googleapis.com`へ接続できる必要があります。
- カメラが別のアプリで使用中の場合は、起動に失敗することがあります。

## 遊び方

1. タイトル画面で「ゲーム開始」を押します。
2. カメラ準備画面でアバター外見を本人が選び、実カメラまたはモック姿勢を選びます。
3. カウントダウン後、壁が画面奥から迫ってきます。
4. 壁が通過するまでに、アバターの判定領域を緑色の安全領域へ収めます。
5. 成功すると壁の`scoreValue`がスコアへ加算されます。安全領域からはみ出すと
   ミス数が増え、ハートが1つ減ります。
6. 壁を何枚か連続して成功すると、壁の速度レベルが段階的に上がります。
7. ハート5個がすべてなくなると、最終スコア、クリア枚数、最高速度、ミス数が表示されます。

アバター外見は「男性風」「女性風」「ニュートラル」から選択できます。カメラ画像から
性別や性自認を推定することはありません。壁は初期状態では約2.4秒で判定位置へ到達し、
成功枚数に応じて段階的に速くなります。安全領域は矩形だけでなく、頭、胴体、腕、
脚の余白をつないだ緑色のシルエットとして表示されます。安全領域の境界には
姿勢検出の揺れを考慮した許容幅があります。現行の壁は上端をゲーム画面の上端へ
固定して迫ります。壁パターンごとに将来別の配置基準を指定できます。

### 実カメラモード

「カメラを開始」を押し、ブラウザのカメラ利用を許可します。プレビューで全身が
映る位置へ移動し、姿勢検出の初期化後に「準備完了」を押します。プレイ中は検出した
姿勢に追従する、壁を向いた後ろ姿アバターがCanvasへ描画されます。肩、腰、足首などを
十分に検出できない場合は未検出として扱われます。

### モック姿勢モード

「モック姿勢で試す」を押すと、カメラやMediaPipeを使用せず、固定の中央姿勢で
ゲームを開始します。壁進行、当たり判定、スコア、ミス、結果画面の流れを確認する
ためのフォールバックです。

- [ ] 実姿勢に追従する後ろ姿アバターと判定領域がCanvasに描画される
- [ ] 緑色の安全領域が四角だけでなく、人型に近い連続したシルエットとして表示される
- [ ] 全身が映らない場合に未検出状態が表示される
- [ ] モック姿勢モードでもカウントダウンからゲームが進行する
- [ ] 壁が進行し、通過時に成功・失敗・未検出のいずれかが1回だけ判定される
- [ ] 壁が滑らかに約2.4秒で迫り、安全領域の許容幅内では成功になる
- [ ] 成功枚数に応じて速度レベルが上がり、速度アップ表示が出る
- [ ] 壁が進行中も上端から離れず、ゲーム画面の上部から迫って見える
- [ ] 主要パターンの緑の安全領域が、頭・胴体・腕・脚を想起できる形になっている
- [ ] 成功時にスコアが増え、失敗時にミス数が増えてハートが1つ減る
- [ ] ハートが5個なくなると結果画面へ遷移する
- [ ] 結果画面に最終スコア、クリア枚数、最高速度、ミス数が表示される
- [ ] 結果画面から「もう一度プレイ」で再開できる
- [ ] 再開時に速度レベルとクリア枚数が初期化される
- [ ] カメラ起動失敗や姿勢検出失敗時に再試行できる

- [ ] タイトル復帰、結果、エラー、リセット時にカメラが停止する
- [ ] 複数回再試行してもタイマーや姿勢検出ループが二重起動しない

## 既知の制限と注意点

- 姿勢検出は1人分のみを対象にします。
- アバター外見は本人が選択し、カメラ画像から属性を推定しません。
- 判定は姿勢ランドマークから作った判定領域の代表点と、壁の複合安全領域による
  簡易判定です。一部の壁は矩形安全領域へフォールバックします。
- 全身、特に肩、腰、足首がカメラに入らない場合は判定不能になります。
- MediaPipeの推論はメインスレッドで同期実行するため、端末性能によっては描画が
  滑らかでない場合があります。
- MediaPipeの外部配信元へ接続できないオフライン環境では、実カメラモードを
  初期化できません。モック姿勢モードを使用してください。
- ランキング、ユーザー登録、スコア保存、バックエンド連携は実装していません。
- ブラウザでカメラを拒否した後は、サイト設定から権限を変更する必要がある場合が
  あります。
