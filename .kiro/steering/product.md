# Product Overview

`steer-clear-of-the-wall` は、カメラの前で体を動かし、画面奥から迫る壁の穴にアバターを合わせて避けるブラウザ向け体験型ゲームです。

プレイヤーのカメラ映像を主表示にはせず、検出した姿勢をゲーム画面上のアバターとして描画します。プレイヤーは自分の体を動かして、壁の安全領域にアバターを収めることを目指します。

## Core Experience

ゲーム体験の中心は次の流れです。

1. Camera Input
2. Pose Detection
3. Avatar Display
4. Wall Pattern
5. Collision Judgment
6. Score / Result

カメラ入力は操作手段であり、画面上の主役はアバター、迫る壁、安全領域、スコアです。カメラ映像をそのまま見せるデモではなく、体を使って遊ぶゲームとして成立させることを重視します。

## Core Capabilities

- カメラからプレイヤーの姿勢を取得する。
- 姿勢検出結果を、直接のカメラ映像ではなくアバター表示へ変換する。
- 複数の壁パターンを出現させ、プレイヤーの姿勢と照合する。
- 人型に近い安全領域を表示し、見た目と判定に同じ形状定義を使う。
- 成功、失敗、スコア、速度上昇、結果を短時間で分かりやすく提示する。
- カメラ権限や姿勢検出失敗時に、ユーザーがリトライできる状態を用意する。
- カメラが使えない環境でも、モック姿勢でゲーム進行を確認できる。

## Target Use Cases

- ブラウザで手軽に試せる体験型ゲーム。
- 展示、授業、イベントなど、短時間で理解して遊べるデモ。
- 姿勢検出、Canvas描画、ゲームロジックを組み合わせた学習用プロジェクト。

## Value Proposition

体をコントローラーにすることで、キーボードやゲームパッドなしに直感的に遊べます。

また、カメラ映像をそのまま表示するのではなくアバター化することで、ゲームらしい見た目とプライバシーへの配慮を両立します。

## Product Principles

- 初回ユーザーが、何をすればよいかを画面遷移と表示だけで理解できるようにする。
- 危険なジャンプ、過度な反り、床に倒れ込む姿勢を要求しない。
- 壁の安全領域は、視認性、納得感、遊びやすさを優先して調整する。
- 実カメラ環境に依存しすぎず、モック姿勢でも主要なゲーム挙動を確認できるようにする。
- スコアや速度上昇は、短時間プレイでも上達や緊張感が分かる粒度にする。
