# Requirements Document

## Introduction

`auto-start-after-calibration` は、実カメラモードのカメラ準備画面で、全身検出と
位置合わせが良好になったら、プレイヤーが「準備完了」をクリックしなくても自動的に
カウントダウンへ進む改善です。

現状では、肩、腰、足首が検出され、中央位置と距離感が良好になった後に、プレイヤーが
「準備完了」を押す必要があります。展示や初回体験では、体を動かして立ち位置を合わせた
直後にクリック操作へ戻ることが流れを止める場合があります。この機能では、良好状態が
短時間安定したことを確認してから自動開始し、体を使うゲーム体験へ自然につなげます。

## 目的

- 実カメラモードで、全身が適切に映ったら自動的にゲーム開始カウントダウンへ進める。
- 一瞬だけ良好状態になった場合の誤開始を避ける。
- 自動開始までの状態をプレイヤーに短く伝え、突然画面が変わったように感じさせない。
- モック姿勢モードと展示向けタイトル復帰導線を維持する。
- カメラ、姿勢検出、ゲーム進行、判定、スコアの既存責務を崩さない。

## 対象範囲

### 今回含める範囲

- 実カメラ準備画面での自動開始条件
- 自動開始までの短い安定待ち時間
- 自動開始予定または開始待ち状態の表示
- 既存の「準備完了」ボタンの扱い変更または補助導線化
- 自動開始判定の純粋ロジックとユニットテスト
- READMEと手動確認チェックリストの更新

### 今回含めない範囲

- モック姿勢モードの自動開始
- プレイ中の当たり判定、壁速度、壁パターン、スコア計算の変更
- カメラ、MediaPipe、Canvas描画方式の大きな変更
- 音声案内、効果音、ランキング、バックエンド連携
- 複数人検出やプレイヤー選択

## Requirements

### Requirement 1: 良好な位置合わせ後の自動開始

**Objective:** As a プレイヤー, I want 全身が適切に映ったら自動でゲームが始まる, so that 体を合わせた後にボタン操作へ戻らず遊び始められる

#### Acceptance Criteria

1. When 実カメラ準備画面でキャリブレーション状態が良好になる, the Auto Start shall 自動開始の待機を開始する
2. When 良好状態が必要な時間だけ継続する, the Auto Start shall 既存のカウントダウン画面へ遷移する
3. If 良好状態ではない, the Auto Start shall カウントダウンへ遷移しない
4. The Auto Start shall モック姿勢モードでは自動開始せず、既存の「モック姿勢で試す」操作を維持する

### Requirement 2: 誤開始防止

**Objective:** As a プレイヤー, I want 一瞬だけ姿勢が検出されても勝手に始まらない, so that 立ち位置調整中に意図せずゲームが始まらない

#### Acceptance Criteria

1. When キャリブレーション状態が良好になってから短時間で不十分な状態へ戻る, the Auto Start shall 自動開始の待機を解除する
2. When 姿勢検出モデルが未初期化である, the Auto Start shall 自動開始しない
3. When カメラストリームが存在しない準備画面である, the Auto Start shall 自動開始しない
4. The Auto Start shall 同じ準備状態からカウントダウン開始を複数回発火しない

### Requirement 3: 自動開始の表示

**Objective:** As a 初回プレイヤー, I want もうすぐ始まることが分かる, so that 画面遷移に驚かず構えられる

#### Acceptance Criteria

1. While 自動開始の待機中である, the Auto Start shall カメラ準備画面に開始予定であることを表示する
2. If 自動開始の待機が解除される, the Auto Start shall 位置合わせ案内へ戻す
3. The Auto Start shall 主要な位置合わせ案内、アバター外見選択、タイトル復帰導線を妨げない
4. The Auto Start shall 小さい画面でも表示や操作が重ならないようにする

### Requirement 4: 手動導線と展示導線の維持

**Objective:** As a 展示運営者, I want 自動開始があってもリセットや確認操作を維持したい, so that 会場運用で詰まらない

#### Acceptance Criteria

1. The Auto Start shall カメラ準備画面の「タイトルへ戻る」導線を維持する
2. When 自動開始待機中に「タイトルへ戻る」を押す, the Auto Start shall タイトルへ戻り、カウントダウンへ進まない
3. The Auto Start shall 結果画面、エラー画面、プレイ中終了の展示運用挙動を変更しない
4. If 手動の準備完了操作を残す場合, the Auto Start shall 良好状態でのみ既存と同じカウントダウンへ進める

### Requirement 5: 責務分離とテスト容易性

**Objective:** As a 開発者, I want 自動開始の判定をUIから分離したい, so that 待機時間や条件を安全に調整できる

#### Acceptance Criteria

1. The Auto Start shall 自動開始の待機時間をテスト可能な定数として管理する
2. The Auto Start shall 自動開始可能かどうかを、ReactやブラウザAPIに依存しない純粋関数で判定できるようにする
3. The Auto Start shall カメラ取得、姿勢検出、Canvas描画、ゲーム判定、スコア計算の既存ロジックを変更しない
4. The Auto Start shall カウントダウンへの遷移には既存の開始処理を再利用する

### Requirement 6: 検証

**Objective:** As a 開発者, I want 自動開始の挙動を自動検証と実機確認で確認したい, so that カメラ準備導線の回帰を防げる

#### Acceptance Criteria

1. The Auto Start shall 自動開始判定のユニットテストを追加する
2. The Auto Start shall 型チェック、ビルド、ユニットテストが成功する状態を維持する
3. The Auto Start shall モック姿勢E2Eの既存導線を壊さない
4. The Auto Start shall 実カメラで、良好状態の継続による自動開始と、調整中に自動開始しないことを手動確認できるようにする
5. The Auto Start shall READMEまたは手動確認チェックリストへ確認観点を反映する

## 手動確認観点

- カメラ許可後、全身が映っていない状態では自動開始しない。
- 全身、中央位置、距離感、安定性が良好になると、自動開始待機の表示が出る。
- 良好状態が短時間続くと、クリックなしでカウントダウンへ進む。
- 自動開始待機中に体を外すと、待機が解除される。
- 自動開始待機中に「タイトルへ戻る」を押すと、タイトルへ戻り、後からカウントダウンへ進まない。
- モック姿勢モードは従来どおりボタン操作で開始する。
- カウントダウン、プレイ、結果、エラー、展示向けリセット導線は従来どおり動作する。
