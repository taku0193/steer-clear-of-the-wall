# Implementation Plan

## 方針

- `safeArea` は互換用の外接矩形として残し、複合形状は `safeShape` として追加する。
- 描画と判定は同じ `safeShape` を入力に使い、見た目と判定のズレを避ける。
- 初期実装では主要パターンを人型化し、残りは矩形フォールバックで維持する。
- 判定は `playerArea` から作る代表点で行い、CanvasやMediaPipeに依存しない純粋ロジックにする。

- [x] 2. 複合形状の内外判定ロジックを追加する

- [x] 3. 衝突判定を複合形状へ対応する
  - 作業内容: `safeShape` がある壁では代表点判定を使い、ない壁では既存矩形判定を維持する。
  - 変更するファイル: `src/game/collision.ts`, `src/game/collision.test.ts`

- [x] 4. Canvas描画を複合形状の穴へ対応する
  - 作業内容: `safeShape` から安全領域パスを作り、赤い壁を人型にくり抜き、緑の輪郭を描く。
  - 変更するファイル: `src/rendering/canvasRenderer.ts`

- [x] 5. 主要壁パターンを人型形状へ移行する
