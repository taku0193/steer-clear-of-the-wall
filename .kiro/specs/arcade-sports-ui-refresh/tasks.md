# Implementation Plan

## 方針

- 共通視覚ルールを先に整え、画面ごとの個別指定を増やさない。
- ゲーム状態、壁パターン、衝突判定、スコア、速度の規則は変更しない。
- タイトルとプレイ画面のCanvasは既存`renderGameCanvas`を再利用する。
- 画面単位で実装と確認を行い、最後にレスポンシブと演出を統合調整する。
- 各段階で型チェックを実行し、Canvasや状態遷移の回帰を早期に検出する。
